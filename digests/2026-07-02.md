## 🔝 Top Picks

- **[AutoMem: Memory as a trainable skill](https://arxiv.org/abs/2607.01224)** — Treats agent memory management (what to encode, when to retrieve, how to organize) as an *independently learnable* skill, optimizing both the memory *structure* (schemas, action vocab) and the model's memory *proficiency* via self-distilled good decisions. Optimizing memory alone — no change to task behavior — gave **~2–4× gains** on long-horizon games and pushed a **32B open-weight model to parity with Claude Opus 4.5 / Gemini 3.1 Pro**. The strongest agentic-memory result of the day.
- **[AutoTrainess: LMs that post-train LMs autonomously](https://arxiv.org/abs/2606.31551)** — An LM agent that exposes planning, data prep, training, eval, and logging as agent-computer interfaces, externalizing human post-training know-how as explicit workflows/rules rather than a raw CLI. Beats CLI-only baselines on PostTrainBench (26.94 vs 23.21 with Codex) and generalizes across models/harnesses. A concrete step toward self-improving training loops.
- **[LOCOS: finding non-literal retrieval heads](https://arxiv.org/abs/2607.01002)** — A *write-aware* interpretability detector that scores attention heads by their OV-circuit projection onto the answer-token direction, catching heads that *synthesize* answers rather than literally copy. Ablating 50 heads on Qwen3-8B collapses non-literal retrieval (ROUGE-L 0.401→0.000) while leaving recall/arithmetic untouched — a clean, mechanism-grounded advance over copy-matching detectors.

## 🧠 Agentic Memory & Personalization

- **[AutoMem](https://arxiv.org/abs/2607.01224)** — (see Top Picks) Memory as a learnable metamemory skill; dual optimization of structure + proficiency. [open-weights base] · [code](https://github.com/) referenced.
- **[MemSyco-Bench: sycophancy in agent memory](https://arxiv.org/abs/2607.01071)** — First benchmark for *memory-induced* sycophancy: retrieved memories often make agents over-align with the user at the cost of factual accuracy. Five tasks probe whether agents can reject memory as fact, respect its scope, resolve memory-vs-evidence conflicts, track updates, and use memory for legit personalization. Names a real failure mode the "store/retrieve correctly" benchmarks miss. [[code](https://github.com/XMUDeepLIT/MemSyco-Bench)]
- **[Personalization as Inverse Planning (SPIRE)](https://arxiv.org/abs/2607.00407)** — Frames page-level slide personalization as inferring latent design intent without knowing the executing tool. Deliberately corrupts clean slides to create a verifiable *structural denoising* surrogate that two RL agents refine collaboratively — with a proof that denoising is a consistent surrogate and that the multi-agent form reduces policy-gradient variance. Nice reframe of a personalization problem as learnable inverse planning.

## 🎯 RL & Post-Training Methods

- **[Right in the Right Way: RLVR + human demonstrations](https://arxiv.org/abs/2607.01181)** — Augments verifiable rewards with a *learned* adversarial discriminator trained to tell human from model outputs, giving signal on the non-verifiable properties (style, structure) that RLVR ignores. Cuts diversity collapse and reward hacking while preserving accuracy — nearly eliminates misbehavior on a reward-hacking benchmark. Bridges RL and SFT in a principled way.
- **[QuasiMoTTo: Quasi-Monte Carlo test-time scaling](https://arxiv.org/abs/2607.01179)** — Replaces i.i.d. parallel samples with *correlated-but-marginally-exact* samples drawn via QMC over the inverse-CDF of autoregressive sampling, so the batch covers the output space with far less redundancy. Matches i.i.d. pass@k with **25–47% fewer samples** and cuts GRPO training steps by 50%. A genuinely new sampler design, drop-in for both inference scaling and RL.
- **[Is One Layer Enough?](https://arxiv.org/abs/2607.01232)** — Challenges the assumption that RL post-training should update all layers uniformly; shows training a *single* transformer layer can match full-parameter RL. A surprising empirical finding about where RL gains actually live in the network.
- **[AutoTrainess](https://arxiv.org/abs/2606.31551)** — (see Top Picks) Agent that autonomously runs the full post-training loop.

## 🤖 Self-Evolving & Robust Agents

- **[Next-Gen Agentic RL Systems (AReaL2.0)](https://arxiv.org/abs/2607.01120)** — Argues the bottleneck for self-evolving enterprise agents is *systems*, not RL algorithms: no standardized step-granular trajectory protocol, no governed data proxy from real workloads, no unified "when to update weights vs. evolve the harness" control plane. Sketches architectures and instantiates one branch (AReaL2.0) as an agent-oriented online RL loop. A useful systems-level framing of continual agent learning.
- **[Can Agents Generalize to the Open World? (OpenAgent)](https://arxiv.org/abs/2607.01084)** — Formalizes open-world tool use with controlled distributional shifts across a Perception→Interaction→Reasoning→Internalization hierarchy, showing both SFT- and RL-trained agents degrade under shift, and proposes perturbation-augmented fine-tuning as a fix. Good diagnostic sandbox for agent robustness. [[code](https://github.com/LAMDA-NeSy/OpenAgent)]
- **[Dockerless: environment-free code verification](https://arxiv.org/abs/2606.28436)** — Verifies agent code patches *without executing them*, judging correctness via agentic repo exploration instead of per-repo Docker images. Beats the strongest open-source verifier by 14.3 AUC and enables a fully environment-free SFT+RL pipeline that matches execution-based post-training on SWE-bench. Removes a major cost/infra bottleneck in coding-agent training.

## 🔬 Interpretability & Safety

- **[LOCOS](https://arxiv.org/abs/2607.01002)** — (see Top Picks) Write-aware detection of non-literal retrieval heads.
- **[The Model Organism Lottery](https://arxiv.org/abs/2607.01033)** — Builds 54 "model organisms" (LMs trained to exhibit hidden behaviors) across seven training techniques and finds interpretability results depend heavily on *how* the organism was trained — and that more realistic *integrated* training yields organisms that are **harder** to interpret than the usual post-hoc SFT ones. Casts serious doubt on current MOs as validity proxies for white-box methods. Important methodological caution for the interpretability field.

## 🏗️ Architecture, Reasoning & Robotics

- **[The State-Prediction Separation Hypothesis](https://arxiv.org/abs/2607.01218)** — Proposes that transformers overload one computation stream for both next-token prediction and storing state for the future, and that *disentangling* the two roles with a two-stream variant improves language modeling. A clean new architectural hypothesis with pretraining evidence.
- **[Message Passing Enables Efficient Reasoning](https://arxiv.org/abs/2607.01077)** — Lets parallel reasoning threads *communicate directly* via lightweight send/receive primitives (with preemption to kill unpromising branches), instead of the transient, non-communicating threads of fork-join. Asymptotically smaller context on Sudoku, better efficiency on 3-SAT, competitive long-context QA — a new parallel-reasoning primitive.
- **[ASPIRE: Agentic skill discovery for robotics](https://arxiv.org/abs/2607.00272)** — NVIDIA continual-learning system that autonomously writes/refines robot control programs (code-as-policy) and compounds them into a reusable, cross-embodiment skill library via evolutionary search. **+77% on LIBERO-Pro under perturbation**, 31% vs 4% zero-shot on unseen long-horizon tasks, plus early sim-to-real transfer. A strong robotics-agent result worth the secondary-tier slot.
